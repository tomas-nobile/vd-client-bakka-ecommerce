# Spec: Home Categories (diseño Contrive)

## Rol

Implementar en el tema WordPress (bakka) la sección **Categories** siguiendo el diseño de referencia Contrive. Usar **Tailwind** para layout y espaciado; **SCSS** en `src/front-page/styles/` para el aspecto visual (círculos, overlay, hover). **No** usar Bootstrap. Icono de flecha: **SVG inline**. Color de acento: **el mismo del tema** (variable existente). Incluir **todas las animaciones** de la sección. Grid **flexible** para N categorías (no fijo a 4).

---

## Objetivo

Replicar la sección "Shop By Categories" del reference: cabecera centrada (subtítulo + título), grid de cards con imagen circular, overlay con icono al hover y título debajo. Cada card enlaza al archive de la categoría WooCommerce. Sin modales; enlaces directos. Animaciones (ej. fade-up) como en el reference.

---

## Referencia de diseño (consultar)

| Qué | Dónde |
|-----|--------|
| HTML y markup | `0design-goal_envato_contrive/index.html` líneas 1059–1126 |
| Estilos desktop | `0design-goal_envato_contrive/assets/css/style.css` bloque `.categories-con` (aprox. 1009–1062) |
| Responsive | `0design-goal_envato_contrive/assets/css/responsive.css` (media queries `.categories-con`) |

Detalle visual a replicar:
- Sección: padding 135px 0 138px (desktop), fondo blanco.
- Cabecera: h6 (subtítulo) + h2 (título), centrado; márgenes 16px y 52px en desktop.
- Card: figure **circular** (255px desktop), imagen con border-radius 100%; hover imagen: opacity 55%, scale(1.1); transición 0.8s ease-in-out.
- Icono overlay: circular, centrado sobre la imagen, color blanco, fondo **accent del tema**; oculto por defecto, visible al hover del card; tamaño ~52px desktop; **usar SVG inline** (flecha derecha), no Font Awesome.
- Debajo de la imagen: h4 con nombre de categoría.
- Responsive: reducir padding, márgenes y tamaños de figure/icono en breakpoints (206px, 200px, icono 50px, 45px, etc.).

### Tipografía (reference Contrive — replicar en `categories.scss`)

Fuentes del reference: **Jost** para títulos (h2, h4); **Archivo** para subtítulo (h6). Colores: `--e-global-color-secondary` = #000000 (títulos); `--e-global-color-accent` = #fb704f (subtítulo; en el tema usar **accent del tema**).

| Elemento | Font family | Tamaño | Line-height | Peso | Color | Otros |
|----------|-------------|--------|-------------|------|--------|-------|
| **Subtítulo (h6)** | Archivo, serif | 14px | 20px | 400 | accent | uppercase, letter-spacing 2.8px |
| **Título sección (h2)** | Jost, serif | 46px | 50px | 700 | secondary/negro | — |
| **Nombre categoría (h4)** | Jost, serif | 20px | 26px | 600 | secondary/negro | — |

Márgenes cabecera (desktop): h6 margin-bottom 16px; h2 margin-bottom 52px. En responsive solo se ajustan los margin-bottom (14px/40px, 10px/30px, 6px/25px, 4px/20px, 15px); no hay override de font-size para h6/h2/h4 en `.categories-con` en el reference, pero se puede escalar ligeramente en móvil (ej. h2 más pequeño) si el diseño lo pide. Implementar estos valores en `categories.scss` (o variables del tema si ya existen Jost/Archivo); si el tema usa otras fuentes globales, al menos respetar tamaño, peso y color para mantener jerarquía visual.

## Decisiones de diseño

| Tema | Decisión |
|------|----------|
| Icono flecha | SVG inline (no Font Awesome). |
| Color acento overlay | Mismo del tema (variable SCSS existente o la usada en popular-products/hero). |
| Animaciones | Incluir todas: fade-up (o equivalente AOS) en cabecera y grid. |
| Cantidad de categorías | Flexible: N categorías; grid adaptable (1/2/4 columnas con Tailwind), no limitar a 4. |

## Archivos a crear o modificar

1. **`src/front-page/components/categories.php`**  
   Mantener `etheme_render_home_categories($attributes)` y `etheme_render_home_category_card($cat)` usando `etheme_get_home_categories()` del helper. Cambiar markup a diseño Contrive:
   - Section con clase contenedora (ej. `categories-con` o BEM) para enganchar SCSS.
   - Cabecera: contenedor centrado, h6 (subtítulo traducible, ej. "Categories"), h2 (título traducible, ej. "Shop By Categories"); atributos/data para animación (ej. `data-aos="fade-up"` si se usa AOS).
   - Grid con Tailwind: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-*` (flexible para N ítems).
   - Por cada categoría: contenedor card (ej. `.categories-box`), dentro: contenedor imagen con figure circular (clase para SCSS), imagen de la categoría (`thumbnail_id` WooCommerce), enlace al term con **SVG inline** (flecha derecha) como overlay, luego h4 con nombre. Enlace = `get_term_link($cat)`; sin modales.

2. **`src/front-page/includes/front-page-index.helpers.php`**  
   No cambiar firma de `etheme_get_home_categories($mode, $ids)`. Añadir helpers solo si hace falta (ej. preparar datos para la card).

3. **`src/front-page/styles/categories.scss`** (CREAR)  
   - Variables: usar acento del tema (ej. variable existente o definir `$categories-accent` alineada al resto del tema); color de texto secundario (negro) para h2 y h4.
   - **Tipografía:** h6: Archivo 14px/20px peso 400, uppercase, letter-spacing 2.8px, color accent. h2: Jost 46px/50px peso 700, color secundario. h4 (nombre categoría): Jost 20px/26px peso 600, color secundario. Cargar fuentes (Google Fonts: Jost, Archivo) si el tema no las tiene; aplicar en selectores `.categories-con .categories_content h6`, `.categories-con .categories_content h2`, `.categories-con .categories-box h4`.
   - Sección: padding, fondo blanco; responsive (100px, 80px, 60px, 40px según breakpoints).
   - Cabecera: h6 y h2 centrados, márgenes según reference (16px / 52px desktop); estilos para animación si aplican.
   - Card: figure circular 255px desktop, border-radius 100%, overflow hidden; img border-radius y transición; hover img: opacity 55%, transform scale(1.1). Icono overlay: circular, centrado (absolute + margin auto), fondo accent, blanco; display none por defecto, block en hover del card; tamaños responsive (52px → 50px → 45px).
   - Importar en `src/front-page/index/style.scss`: `@import '../styles/categories';`

4. **Animaciones**  
   Incluir todas las animaciones de la sección del reference (ej. fade-up en cabecera y en el grid). Si el tema ya usa AOS: añadir `data-aos="fade-up"` (y opcionalmente `data-aos-delay`) en los nodos correspondientes del markup en `categories.php` y asegurar que el script AOS se cargue en front page si no está global. Si no hay AOS: implementar equivalente (CSS animation o script ligero) para entrada en vista.

## Reglas del proyecto (respetar)

- **README:** Componentes en `src/front-page/components/`; estilos en `src/front-page/styles/*.scss` importados desde el `style.scss` del block. Front-page index es orchestrator; `render.php` ya llama a `etheme_render_home_categories($attributes)`.
- **Tailwind** para layout (container, grid, gap, padding, text-center). **SCSS** para diseño Contrive (círculos, overlay, hover, tamaños exactos).
- **i18n:** todos los textos visibles con `esc_html_e()` / `__()` dominio `etheme`.
- **Accesibilidad:** section con `aria-labelledby` si hay id en el título; `alt` en imágenes; enlace del icono con texto o `aria-label` si solo hay SVG.
- **Seguridad:** `esc_url()`, `esc_attr()`, `esc_html()` en salidas dinámicas.
- No ejecutar `npm run build` (regla del proyecto).
