# Spec: Popular Products (diseño Contrive)

## Rol

Implementar en el tema WordPress (tema bakka) la sección "Featured Items / Our Popular Products" siguiendo el diseño de referencia Contrive. Usar **Tailwind** para layout y espaciado; **SCSS** en `src/front-page/styles/` para el aspecto visual. **No** usar Bootstrap.

---

## Objetivo

Replicar la sección de productos populares con tabs por categoría y cards con imagen, color dots dinámicos, título y precios. La apariencia debe seguir el reference; el detalle visual está en los archivos indicados abajo.

---

## Referencia de diseño (consultar estos archivos)

| Qué | Dónde |
|-----|--------|
| Estructura HTML y markup | `0design-goal_envato_contrive/index.html` líneas 372–990 |
| Estilos desktop | `0design-goal_envato_contrive/assets/css/style.css` bloque `.feature-con` (aprox. 712–882) |
| Estilos responsive | `0design-goal_envato_contrive/assets/css/responsive.css` (media queries `.feature-con`) |

Replicar estructura (sección, título pequeño + título principal, tabs, tab panels, grid de cards) y aspecto (tabs con borde inferior y estado activo, cards con imagen, color dots, precios). Tabs en el tema ya funcionan con `home-popular-products.product-tabs.js` (`data-tab`, `id` de panel); adaptar solo markup y clases al nuevo diseño.

---

## Alcance

**Incluir**
- Tabs por categoría (All + categorías WooCommerce) con estilo Contrive (subrayado activo/hover).
- Grid: 3 columnas desktop, 2 tablet, 1 móvil (Tailwind).
- Card de producto: imagen, **color dots dinámicos** (colores desde atributos/variaciones del producto en WooCommerce), título enlace al producto, precios (actual + tachado si hay oferta).
- Animaciones del reference (ej. fade-up en títulos y grid; AOS o equivalente).
- Empty state: si una categoría no tiene productos, mostrar "No hay productos en esta categoría." y mantener el panel visible.

**No incluir en esta fase**
- Overlay de iconos (carrito, ojo, flecha) sobre la imagen.
- Icono corazón (wishlist).

---

## Componente nuevo: Popular Products Card

- **Archivo:** `src/front-page/components/popular-products-card.php`
- **Función:** p. ej. `etheme_render_home_popular_product_card( $product )` — recibe un `WC_Product`, no hace queries; renderiza una card (imagen, color dots desde atributos, título, precios).
- **Uso:** `popular-products.php` debe llamar a esta función por cada producto en el loop, no implementar la card inline.
- **Carga:** el componente se carga desde `render.php` (ver lista de archivos).

---

## Archivos a crear o modificar

1. **`src/front-page/index/render.php`**  
   Añadir `'popular-products-card'` al array `$components` para que se haga `require_once` de `popular-products-card.php`.

2. **`src/front-page/components/popular-products-card.php`** (CREAR)  
   Función que renderiza una card: imagen, color dots dinámicos (atributos/variaciones WooCommerce), título, precios. Sin overlay ni corazón.

3. **`src/front-page/components/popular-products.php`**  
   Mantener lógica actual (`etheme_render_home_popular_products`, `etheme_get_popular_products()`). Cambiar markup: sección con clase para SCSS (ej. `popular-products-section`), título pequeño + título principal, tabs con `role="tablist"`/`role="tab"` y `data-tab`, paneles con `id="products-panel-all"` e `id="products-panel-{slug}"`, `role="tabpanel"`. Grid con Tailwind (3/2/1 columnas). En cada panel, por cada producto llamar a la función del card. Mantener accesibilidad (aria-selected, aria-controls, aria-labelledby).

4. **`src/front-page/includes/front-page-index.helpers.php`**  
   No cambiar firma de `etheme_get_popular_products()` ni `etheme_get_home_categories()`. Añadir helper(s) solo si hace falta para obtener colores del producto (atributos/variaciones) para los color dots.

5. **`src/front-page/scripts/home-popular-products.product-tabs.js`**  
   Mantener `initProductTabs()` y lógica por `data-tab` e `id` de panel. Ajustar clases para tab activa y visibilidad de paneles según el nuevo markup. Sin jQuery ni Bootstrap.

6. **`src/front-page/styles/popular-products.scss`** (CREAR)  
   Estilos de sección, títulos, tabs, cards (contenedor imagen, color dots, precios), animaciones y responsive según el reference. Importar en `src/front-page/index/style.scss`: `@import '../styles/popular-products';`.

---

## Restricciones

- No usar Bootstrap. No ejecutar `npm run build`.
- Preferir Tailwind para layout/espaciado; SCSS para lo que no cubra Tailwind (tabs, dots, precios, animaciones).
- Traducciones: dominio `etheme`. Atributos del bloque existentes: respetar.
- Accesibilidad y semántica: mantener (ARIA, roles, ids).
