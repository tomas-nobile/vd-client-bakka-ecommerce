### Detección del ítem «Tienda»

El desplegable con categorías/subcategorías debe aplicarse **solo** al ítem de menú que representa la tienda. Implementar en este orden (primera coincidencia gana):

1. **Coincidencia con la página Tienda de WooCommerce:** si el ítem del menú es de tipo «página» y su `object_id` es `wc_get_page_id( 'shop' )`, tratarlo como Tienda.
2. **Clase CSS personalizada:** si en el ítem (Screen Options → «Clases CSS») se añade `etheme-navbar-shop`, tratarlo como Tienda (útil para enlaces personalizados o si la coincidencia por ID no aplica).

El **href** del ítem sigue siendo el de la página Tienda (o el enlace que el editor configuró); el submenú visual es el árbol generado por PHP, no hijos del menú en la base de datos.

### Reglas del README que aplican

- `import './style.scss'` desde `index.js` del bloque (no solo `edit.js`).
- No ejecutar `npm run build` en trabajo de agente; asumir `npm run start:watch` fuera de banda.
- Registrar bloque en `functions.php`; si el bloque se usa en plantillas/parts, enqueue según patrones documentados en README.
- Escape: `esc_html`, `esc_url`, `esc_attr`.

## Prompt (copiar para el agente de código)

**Rol:** Eres un desarrollador senior del tema WordPress FSE **Bakka** (`wp-content/themes/bakka/`, WSL). Implementa el bloque de navegación en `src/core/navbar/` según el README y esta spec.

### Objetivo

Replicar la **composición y jerarquía** del navbar Contrive (`0design-goal_envato_contrive/index.html`, ~70–151): logo a la izquierda, grupo de enlaces al estilo `navbar-nav` + `ml-auto`, bloque de acciones a la derecha (`last_list`: búsqueda, carrito con contador si aplica, cuenta). **No** portar Bootstrap: usa **Tailwind + SCSS** del bloque para equivalentes visuales.

### Menú WordPress (opción A)

- Registra una **theme location** y usa `wp_nav_menu` con un `Walker` personalizado o filtros (`nav_menu_css_class`, `nav_menu_item_id`, `walker_nav_menu_start_el`) según necesites para:
  - Marcar el ítem Tienda y renderizar su **submenú como árbol de `product_cat`** (no como `<ul>` de hijos del menú).
  - Mantener el resto de ítems como enlaces normales y dropdowns estándar si el menú los tiene.

### WooCommerce — árbol automático

- Una consulta eficiente: `get_terms( [ 'taxonomy' => 'product_cat', 'hide_empty' => true, 'parent' => 0 ] )` y agrupar hijos por `parent`, o `get_terms` jerárquico con orden; **evitar** una query por categoría.
- Cache: `set_transient` / `wp_cache_get` con invalidación en `created_product_cat`, `edited_product_cat`, `delete_product_cat` (o acciones WooCommerce equivalentes) con TTL razonable (p. ej. 12–24 h) o invalidación explícita.
- Enlaces: `get_term_link( $term )`; si no hay términos, degradar sin romper el layout (mensaje mínimo o solo enlace a tienda).

### Visual

- **Navbar negra**; tipografía: extraer del CSS de referencia **tamaños, pesos, color, spacing** entre items y alinear con Tailwind/utilidades + tokens SCSS si hace falta.
- **Moderno:** transiciones en hover, apertura de submenús desktop, estados `:focus-visible`.
- **Móvil:** overlay fullscreen para el menú; botón cerrar; animaciones; foco atrapado de forma razonable en el panel abierto.

### Búsqueda

- Icono abre **modal overlay** con campo de búsqueda; cerrar con botón, Escape; backdrop semitransparente; accesibilidad (`aria-modal`, `role="dialog"`, título oculto si hace falta).

### Assets

- Imágenes desde **`get_theme_file_uri( 'assets/images/...' )`** (logo, iconos header). No referenciar rutas bajo `0design-goal_envato_contrive`.

### Editor

- `block.json` / `edit.js`: atributos mínimos — por ejemplo **ubicación del menú** (slug de theme location), opcional logo override, toggles para iconos.
- Placeholder claro si no hay menú asignado.

### Entregables

- Árbol completo bajo `src/core/navbar/` (orquestador + componentes + scripts + estilos).
- Registro en `functions.php`.
- Sin refactors ajenos al navbar; sin añadir Bootstrap como dependencia del tema.

### No hagas

- Duplicar contenido de otras specs en README; si el comportamiento del tema cambia de forma relevante, una línea en README bajo la tabla de specs puede bastar.

## Criterios de aceptación (resumen)

- [ ] Menú editable desde WP; ítem Tienda detectado por ID de página shop **o** clase `etheme-navbar-shop`.
- [ ] Dropdown de Tienda muestra categorías y subcategorías desde `product_cat` sin configuración manual de cada enlace.
- [ ] Navbar oscura; desktop fiel en ritmo al mockup; móvil fullscreen + cerrar + animaciones.
- [ ] Búsqueda en modal overlay accesible.
- [ ] Estilos cargados en frontend vía `index.js` + `style.scss`; convenciones `etheme_*` y escape aplicados.
