# 16 — Página Contact (Contrive → tema Bakka)

Documento de especificación y **prompt de implementación** (optimizado para Claude Sonnet). Leer `README.md` del tema antes de codificar.

## Decisiones por defecto

- Grupo del bloque: `src/contact/` (orquestador en `src/contact/index/`).
- Formulario: markup + validación básica en cliente; backend de envío (`wp_mail` / REST / AJAX) en fase posterior salvo que se indique lo contrario.
- Imágenes: mismos nombres que en Contrive (`contactinfo-icon1.png`, etc.) bajo una ruta en el tema (p. ej. `assets/images/…`); usar `get_theme_file_uri()` y ajustar la subcarpeta real.
- Plantilla: página estándar con el bloque en el contenido del editor; no es obligatorio crear `templates/page-contact.html` salvo necesidad FSE.

---

## Prompt (copiar para el agente de código)

**Rol:** Desarrollador del tema WordPress FSE **Bakka** (`wp-content/themes/bakka/`). Replicar visual y estructuralmente la página de contacto del HTML de referencia **sin** subbanner, header, footer ni demás chrome del mockup.

### Referencias obligatorias

1. `0design-goal_envato_contrive/contact.html` — únicamente:
   - **Contact Info** (`.contactinfo-con`)
   - **Contact Form** (`.contactform-con`)
   - **Map** (`.map-con`)
2. **Excluir por completo:** `.sub_banner`, `.header`, formulario de búsqueda, footer, preloader, botón back-to-top, scripts de carousel, dropdown país, etc.
3. `0design-goal_envato_contrive/assets/css/style.css` y `responsive.css` — reglas de contacto, `.primary_btn`, tipografía global (h2–h6, `.text-size-*`).
4. `README.md` del tema — orquestador, imports de estilos (`import './style.scss'` en `index.js` del bloque, no solo en `edit.js`), convenciones (`etheme_*`, escape), **no ejecutar** `npm run build` en trabajo de agente.

### Diseño y tokens

- **Fuentes:** body **Archivo**; títulos **Jost** (Google Fonts como en el CSS de referencia).
- **h6 (eyebrow):** 14px, weight 400, uppercase, `letter-spacing: 2.8px`, color accent.
- **h2:** 46px / 700, Jost, color secundario/negro según referencia.
- **h4 (tarjetas):** 20px / 600, Jost.
- **Colores:** accent `#fb704f`, texto `#6a6a6a`, fondo sección formulario `#f2f9ff`, blanco `#fff`, fondo círculo icono `#fafafa`, hover botón oscuro `#2c5858`, error `#ff0000`.
- **Contact info:** padding desktop ~135px 0 140px; tarjetas sombra `0 1px 58px 1px rgb(0 0 0 / 3%)`, padding ~46px 50px; icono 87×87px, circular, hover `translateY(-5px)`; enlaces hover accent.
- **Formulario:** mismos paddings de sección; inputs ~53px alto, textarea ~188px; sombra/borde/focus como en `style.css`; `.primary_btn` uppercase, Archivo 16px/600, accent → hover cyan oscuro; márgenes responsive según `responsive.css`.
- **Mapa:** padding ~140px 0; iframe 100%, altura ~547px; grayscale por defecto, color en hover.

### Bootstrap → Tailwind

Usar **Tailwind CSS 4.x** para grid, spacing y breakpoints donde el HTML original usa `.container` / `.row` / `.col-*`. Conservar clases semánticas del diseño (`.contactinfo-con`, `.contact-box`, etc.) y completar detalle visual en SCSS.

### Arquitectura del bloque

- Namespace: `etheme/*`; text domain: `etheme`.
- Orquestador: `src/contact/index/` — `block.json`, `render.php`, `edit.js`, `index.js`, `view.js`, `style.scss`, `editor.scss`.
- `render.php` solo carga componentes; sin HTML masivo en el orquestador.
- Componentes PHP en `src/contact/components/` (sugeridos): `contact-info.php`, `contact-form.php`, `contact-map.php` — funciones `etheme_render_contact_*()`.
- **Estilos:** solo en `src/contact/styles/*.scss`, importados desde `src/contact/index/style.scss`. **No** colocar `.scss` junto a los `.php` de componentes.
- Registrar el bloque en `functions.php` como el resto del tema. Si solo se usa en plantilla FSE sin contenido de post, valorar enqueue manual (véase README: cart/checkout).

### Imágenes

Mismas que el mockup; rutas con `esc_url( get_theme_file_uri( 'assets/…' ) )` (ajustar subcarpeta real en el tema).

### Animación fade-up

Reutilizar el patrón del tema:

- Importar `initFadeUp` desde `src/front-page/scripts/fp-fade-up.js` en `view.js` del bloque, pasando el selector del wrapper del bloque (clase raíz de `.wp-block-etheme-contact-index` o la que defina `block.json`).
- Marcar con `data-aos="fade-up"` los mismos bloques que en el HTML original (títulos, fila de cajas, contenedor del formulario).
- En `style.scss` del bloque, duplicar las reglas de fade-up scoped al **selector del bloque contact** (opacity/transform inicial, `.fp-aos-visible`, transición ~0.55s), con `@media (prefers-reduced-motion: reduce)` sin animación perjudicial.

### Formulario

Campos: Name, Phone, Email, Message, botón «Send Now». No replicar captcha/jQuery validate del mockup salvo backend acordado. Validación: JS alineado al tema (vanilla preferido). Si hay envío: nonces, sanitización, `wp_mail` o endpoint acordado.

### Editor (`edit.js`)

Atributos editables: títulos/subtítulos de secciones, textos y enlaces de las tres cajas (dirección, teléfonos, emails), `src`/URL del iframe del mapa, etiqueta del botón.

### Seguridad

`esc_html`, `esc_url`, `esc_attr` en salidas; nonces en envíos de formulario.

### No hacer

Subbanner ni breadcrumbs hero del mockup; navbar/footer del HTML; copiar jQuery plugins del paquete Contrive sin necesidad; documentación nueva fuera de este spec/README según reglas del proyecto.

### Verificación

- `index.js` del bloque importa `./style.scss`.
- CSS compilado incluye estilos de contacto.
- Fade-up al hacer scroll dentro del bloque.
- Contraste y foco visible en campos.

## Criterios de aceptación (resumen)

- [ ] Tres secciones (info, formulario, mapa) visualmente alineadas con Contrive en desktop y breakpoints clave.
- [ ] Tipografía y colores coinciden con tokens anteriores (tolerancia mínima por Tailwind/theme.json).
- [ ] Sin subbanner ni chrome del HTML de referencia.
- [ ] Bloque registrado, componentes PHP separados, SCSS en `styles/`.
- [ ] Fade-up reutilizando `fp-fade-up.js` y CSS scoped al bloque.
- [ ] Cumplimiento README (carga de estilos en frontend, escape).
