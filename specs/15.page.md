# Spec 15 — Página «Mi cuenta» (WooCommerce)

---

## Prompt ejecutable (Claude Sonnet) — copiar desde aquí

**Role:** Senior WordPress 6.x + WooCommerce + FSE block theme engineer.

**Repository / theme:** `bakka` (slug `bakka`). Block namespace `etheme/*`, text domain `etheme`. Workspace: theme root under `wp-content/themes/bakka`. Do not add new `.md` files besides maintaining this spec if needed.

**Objective:** Implement or complete the **My Account** experience as a **new dynamic block** `etheme/page-index` with PHP components and SCSS, visually matching the Envato **Contrive** auth screens (login + register/join + logged-in dashboard shell), while **keeping the site header and footer** (template is not “blank”).

**You must create (if missing):**

1. **Orchestrator block** under `src/page/index/`: `block.json`, `render.php`, `index.js` (with `import './style.scss'`), `edit.js`, `editor.scss`, `view.js`.
2. **PHP components** under `src/page/components/*.php` — reusable render functions (e.g. `etheme_render_*`); **no** large HTML blobs in `render.php` beyond minimal wrappers.
3. **Styles** under `src/page/styles/*.scss`, imported from `src/page/index/style.scss`. No `*.scss` next to individual PHP component files.
4. **Registration:** `functions.php` → `register_block_type( __DIR__ . '/build/page/index' )`.
5. **Webpack:** entries `page/index/index` → `src/page/index/index.js` and `page/index/view` → `src/page/index/view.js`.
6. Use **`src/page`** everywhere; **never** `page-myaccount` (legacy). Remove stale `build/page-myaccount/` after a successful compile if it still exists.

**Design reference (pixel- and token-faithful):**

- HTML: `0design-goal_envato_contrive/login.html`, `0design-goal_envato_contrive/join-now.html`
- CSS: `0design-goal_envato_contrive/assets/css/style.css` — sections *Login Form Page Style* and *Join-Now Form* (~3990–4105+), plus `:root` color variables.
- Reference uses **Bootstrap**; **do not enqueue Bootstrap** in the theme. Use **Tailwind 4 utilities in markup** and **SCSS** for exact values: background `#f7fcf4`, card white, **no border-radius** on card/inputs/button, login column **max-width 450px**, register + logged-in shell **max-width 700px**, section **min-height 100vh**, register extra vertical padding ~**70px** (md+). Inputs: ~16px, weight 500, height **54px**, horizontal padding **21px**, border `1px solid rgb(24 26 23 / 10%)`, focus border accent `#fb704f`. Primary button: full width, ~18px, weight 500, padding ~**20px 15px**, background `#fb704f`, white text, **transition ~0.8s ease-in-out**, hover background `#2c5858`. Links (forgot password, footer): black `#000`, ~16px, underline on hover.
- **Fonts:** load **Jost** (titles; match reference `h2` scale e.g. clamp toward **46px**, line-height ~**50px**, weight **700**, black) and **Archivo** (body) — e.g. Google Fonts `@import` in SCSS scoped to the block wrapper.

**WooCommerce (real behavior, not static HTML):**

- Inside the white card, output the real My Account flow via **`WC_Shortcode_My_Account::output( array() )`** if callable; else `do_shortcode( '[woocommerce_my_account]' )`.
- Preserve notices, nonces, endpoints, and accessible labels.
- Contrive-style titles (**Welcome Back !**, register title, lost password, logged-in greeting) live **outside** the card. Hide **only for guests** any redundant WC **direct child** `h2` of `.woocommerce` that duplicates those titles; **do not** break headings inside logged-in account content (use a modifier class on the block wrapper, e.g. `page-account-block--logged-in`, and scope SR-only / visually hidden rules accordingly).
- Footer links: from login, “Join now…” → `?action=register` when registration is enabled; from register, “Already have an account?” → base my-account URL. Hide or avoid duplicate WC register link inside the login form (`:has(a[href*="action=register"])` or equivalent, with a safe fallback).

**Images:** Use theme paths the team already copied (e.g. `assets/images/logo.png` via `get_theme_file_uri`). If missing, omit logo without breaking layout.

**Fade-up animation:** In `view.js`, import **`initFadeUp`** from `src/front-page/scripts/fp-fade-up.js` and call it with the block wrapper selector (e.g. `.page-account-block`). Add `data-aos="fade-up"` and optional `data-aos-delay` on header, card, and footer link blocks. In SCSS, scope the same opacity/translate + `.fp-aos-visible` pattern as `src/page-cart/index/style.scss` or `src/page-checkout/index/style.scss`, honoring `prefers-reduced-motion`.

**Constraints (read `README.md` “AI Agent Instructions”):**

- Escape output: `esc_html`, `esc_url`, `esc_attr`.
- Do **not** run `npm run build` unless the user explicitly asks; assume **`npm run start:watch`** for compilation.
- Editor must show a clear placeholder; front styles must load from `index.js`, not only `edit.js`.

**Acceptance:** Block appears in editor; guest login/register and lost password match Contrive visually; logged-in navigation + content share the same background/card/typography language; fade-up works without loading AOS JS; no Bootstrap in theme.

## Spec extendida (español) — referencia

### Qué hay que crear

| Pieza | Ruta |
|--------|------|
| Bloque (orquestador) | `src/page/index/` |
| Componentes PHP | `src/page/components/*.php` |
| Estilos | `src/page/styles/*.scss` → import en `src/page/index/style.scss` |

### Plantilla y contenido

- **`templates/page.html`**: página estándar **con** header/footer; el bloque `etheme/page-index` va en el contenido de la página «Mi cuenta» de WooCommerce.

### Criterios de aceptación (resumen)

1. Orquestador + componentes + SCSS alineados con Contrive (fonts, pesos, colores, spacing).
2. WooCommerce funcional (shortcode / clase oficial).
3. Fade-up reutilizando `fp-fade-up.js`.
4. Sin Bootstrap en el tema; Tailwind + SCSS.

### Notas

- Hasta compilar, puede no existir `build/page/index`; es normal hasta ejecutar el watcher de `@wordpress/scripts`.
