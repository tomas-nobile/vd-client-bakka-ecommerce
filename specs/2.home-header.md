## Objetivo

1. **Reemplazar el hero actual con este banner en `hero.php`**  
   - Analiza cómo está construido actualmente `src/front-page/components/hero.php` y **reemplaza su contenido** para que renderice este banner/carousel como el nuevo hero por defecto de la home.  
   - Mantén o adapta las clases (`banner-con`, `banner_content`, `banner_wrapper`, etc.) según el sistema de estilos actual.  
   - Si encaja con el patrón del tema, haz que el contenido textual (títulos, subtítulo, CTA, porcentaje de descuento) sea fácilmente editable desde PHP (variables, argumentos del componente o funciones de WordPress), en lugar de dejarlo completamente hardcodeado.

2. **Integración con `render.php`**  
   - Asegúrate de que `src/front-page/index/render.php` siga montando el hero correctamente en la portada y que ahora utilice el nuevo markup del banner.  
   - Ajusta llamadas, includes o hooks necesarios para que el nuevo hero se muestre correctamente.

3. **Migrar estilos a `hero.scss`**  
   - Añade o ajusta reglas SCSS en `src/front-page/styles/hero.scss` para replicar el diseño original del banner:  
     - Layout de columnas, posicionamiento relativo, imágenes laterales, círculo de oferta (`circle-text`), etc.  
     - Estilos responsivos para desktop, tablet y mobile, reutilizando el sistema de breakpoints ya existente en el proyecto.  
   - Si hay estilos relacionados en otros SCSS (por ejemplo `banner-con.scss`), reutilízalos o unifícalos para evitar duplicaciones.

4. **JavaScript / Carousel**  
   - El HTML original usa un carousel tipo Bootstrap (`.carousel`, `data-ride="carousel"`, `carousel-control-prev/next`, etc.).  
   - Revisa cómo se cargan los scripts en el tema (por ejemplo en `functions.php`) y:  
     - Reutiliza Bootstrap si ya está encolado, **o**  
     - Implementa el comportamiento mínimo del slider con JavaScript propio del tema.  
   - Asegúrate de que los selectores JS apunten al nuevo markup, si cambias IDs o clases.

5. **Rutas de imágenes y assets**  
   - Cambia todas las rutas relativas como `./assets/images/...` o `assets/images/...` para que apunten correctamente a la carpeta `assets/images` del tema en WordPress, usando `get_template_directory_uri()` u otros helpers existentes en el proyecto, por ejemplo:  
     - `<?php echo get_template_directory_uri() . '/assets/images/banner-image.png'; ?>`  
   - Asume que todos los PNG/JPG del banner ya están disponibles dentro de `assets/images/` del tema.

6. **AOS / animaciones**  
   - El HTML incluye atributos `data-aos="fade-up"`. Mantén estos atributos en el nuevo markup para conservar la compatibilidad con la librería AOS, si ya está integrada en el tema.
